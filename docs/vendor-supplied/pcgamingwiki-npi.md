_**Nvidia Profile Inspector**_ (NPI) is an open source third-party tool created for pulling up and editing application profiles within [the Nvidia display drivers](https://www.pcgamingwiki.com/wiki/Nvidia_(GPU) "Nvidia (GPU)"). It works much like the **Manage 3D settings** page in the [Nvidia Control Panel](https://www.pcgamingwiki.com/wiki/Nvidia_Control_Panel "Nvidia Control Panel") but goes more in-depth and exposes settings and offers functionality not available through the native control panel.

The tool was spun out of the original [Nvidia Inspector](http://www.guru3d.com/files-details/nvidia-inspector-download.html) which featured an overclocking utility and required the user to create a shortcut to launch the profile editor separately.

_Nvidia Profile Inspector_ typically sees maintenance updates released every couple of months, keeping it aligned with changes introduced in newer versions of the drivers.

### First steps for tweaking driver settings • [Link](https://www.pcgamingwiki.com/wiki/Nvidia_Profile_Inspector\#First_steps_for_tweaking_driver_settings "Link")

| Select (or create if missing) the display driver profile for the game |
| --- |
| 1. Download and run the latest version of the Profile Inspector, see [Installation](https://www.pcgamingwiki.com/wiki/Nvidia_Profile_Inspector#Installation).<br>2. Using the **Profiles** list on top of the window, select the game-specific profile if one exists. If one does not exist, create a new one:<br>   <br>   1. Click on the **Create new profile** icon (the sun).<br>   2. Give the profile a relevant name. The name of the game is often used.<br>   3. Click on the **Add application to current profile** icon (a small window with an icon of binoculars with a "+" above it).<br>   4. Navigate to [<path-to-game>](https://www.pcgamingwiki.com/wiki/Glossary:Game_data#Installation_folder "Glossary:Game data") and select the game executable and click **Open**.<br>      <br>      - If you get a message that `Filename.exe` already belongs to a profile, verify that the application does not exist in the **Profiles** list again, then re-select the executable again with the file format in the selection dialog set to `Application Absolute Path (*.exe)` instead of the default `Application EXE Name (*.exe)`.<br>3. Tweak the desired settings.<br>4. Finally click on **Apply changes** to save the changes. |

### [Anisotropic filtering (AF)](https://www.pcgamingwiki.com/wiki/Glossary:Anisotropic_filtering_(AF) "Glossary:Anisotropic filtering (AF)") • [Link](https://www.pcgamingwiki.com/wiki/Nvidia_Profile_Inspector\#Anisotropic_filtering_.28AF.29 "Link")

[![Example of a global override.](https://thumbnails.pcgamingwiki.com/8/8e/Nvidia_Profile_Inspector_Anisotropic_Filtering_Override.png/300px-Nvidia_Profile_Inspector_Anisotropic_Filtering_Override.png)](https://www.pcgamingwiki.com/wiki/File:Nvidia_Profile_Inspector_Anisotropic_Filtering_Override.png)

Example of a global override.

Some developers' own texture filtering solutions might be less than ideal, e.g.: the game doesn't support 16x anisotropic filtering either in-game or through configuration files; the game supports 16x anisotropic filtering but doesn't apply it equally on all surfaces; the game supports 16x anisotropic filtering and applies it on all surfaces but texture clarity is somehow not as good as it should be. [This](http://images.nvidia.com/geforce-com/international/comparisons/just-cause-3/just-cause-3-nvidia-control-panel-anisotropic-filtering-interactive-comparison-001-on-vs-off-rev.html) is an example of why it may be relevant to force high quality texture filtering globally.

There is a risk of negative side effects occurring in some games as a result of a forced global driver-level override, with unexpected consequences ranging from textures looking wrong to some post-processing shaders being affected if they rely of a texture that have an unintended level of filtering applied to it. If Nvidia becomes aware of a game that has a conflict with driver forced texture filtering, they may set a special flag on the game profile for texture filtering that disables driver overrides for that game, such as with [Far Cry 3](https://www.pcgamingwiki.com/wiki/Far_Cry_3 "Far Cry 3") and [Doom (2016)](https://www.pcgamingwiki.com/wiki/Doom_(2016) "Doom (2016)"), though this is not a guarantee as proper in-depth compatibility testing is impossible to perform for all games.[\[2\]](https://www.pcgamingwiki.com/wiki/Nvidia_Profile_Inspector#cite_note-2)

A quicker way of doing this is to use [Nvidia Control Panel](https://www.pcgamingwiki.com/wiki/Nvidia_Control_Panel "Nvidia Control Panel") and set **Anisotropic filtering** to `16x` and **Texture filtering - Quality** to `High quality` under **Manage 3D settings** \> **Global Settings**, but it won't cover all specific settings.

| Force highest quality texture filtering globally |
| --- |
| 1. Download and run the latest version of the Profile Inspector, see [Installation](https://www.pcgamingwiki.com/wiki/Nvidia_Profile_Inspector#Installation).<br>2. Select the `_GLOBAL_DRIVER_PROFILE (Base Profile)` profile.<br>3. Set the relevant settings under **Texture Filtering** as shown below ( [or in this image](https://www.pcgamingwiki.com/wiki/File:Nvidia_Profile_Inspector_Anisotropic_Filtering_Override.png "File:Nvidia Profile Inspector Anisotropic Filtering Override.png")):<br>   <br>   - **Texture filtering - Anisotropic filter optimization** to `Off`.<br>   - **Texture filtering - Anisotropic sample optimization** to `Off`.<br>   - **Anisotropic filtering mode** to `User-defined / Off`.<br>   - **Anisotropic filtering setting** to `16x`.<br>   - **Texture filtering - Driver Controlled LOD Bias** to `On`.<br>   - **Texture filtering - Negative LOD bias** to `Clamp`.<br>     <br>     - Setting **Texture filtering - Negative LOD bias** to `Clamp` can lower the quality of DLSS and other [Reconstruction Methods](https://www.pcgamingwiki.com/wiki/Glossary:Anti-aliasing_(AA)#Reconstruction_methods "Glossary:Anti-aliasing (AA)") that rely on a negative LOD bias.\[ _citation needed_\]<br>   - **Texture filtering - Quality** to `High quality`.<br>   - **Texture filtering - Trilinear optimization** to `Off`.<br>4. Press **Apply changes** in the top-right corner. |

| Force anti-aliasing for a select game |

Set **Antialiasing (MSAA) - Behaviour Flags** to `None`.<br>7. Set **Antialiasing (MSAA) - Mode** to `Override any application setting` or `Enhance the application setting` depending on the flag in use, desired AA type, and game in question.<br>8. Set **Antialiasing (MSAA) - Setting** to a setting between `2x [2x Multisampling]` and `32xS [Combined: 2x2 SS + 8x MS]` depending on the flag in use, desired AA type, and game in question.<br>9. Press **Apply changes** in the top-right corner. |

### Layer Vulkan/OpenGL on DXGI swapchain • [Link](https://www.pcgamingwiki.com/wiki/Nvidia_Profile_Inspector\#Layer_Vulkan.2FOpenGL_on_DXGI_swapchain "Link")

Layering Vulkan and OpenGL games on a DXGI swapchain allows for improved multitasking and using [Auto HDR](https://www.pcgamingwiki.com/wiki/Glossary:High_dynamic_range_(HDR) "Glossary:High dynamic range (HDR)") on Windows 11.[\[3\]](https://www.pcgamingwiki.com/wiki/Nvidia_Profile_Inspector#cite_note-3)

| Instructions:[\[4\]](https://www.pcgamingwiki.com/wiki/Nvidia_Profile_Inspector#cite_note-4) |
| --- |
| 1. Launch [Nvidia Control Panel](https://www.pcgamingwiki.com/wiki/Nvidia_Control_Panel "Nvidia Control Panel") and navigate to **Manage 3D settings**.<br>2. Locate the **Vulkan/OpengL present method** (typically at the bottom).<br>3. Change it to `Prefer layered on DXGI Swapchain`.<br>4. Press **Apply** to save the changes.<br>5. Launch **Nvidia Profile Inspector** v2.4.0.19 (March 20, 2025) or later.<br>6. In the section called **2 - Sync and Refresh _, locate the_** _Vulkan/OpenGL Present Method - Flags'_ setting.<br>7. Change it to `0x000802A5` by double-clicking on the value field and copy/pasting the value.<br>   <br>   - This value will enable the DXGI swapchain layer in most scenarios, including [DXVK](https://www.pcgamingwiki.com/wiki/DXVK "DXVK").<br>8. Press **Apply changes** in the top-right corner. |

### Prevent Ansel from being injected/enabled for a specific game • [Link](https://www.pcgamingwiki.com/wiki/Nvidia_Profile_Inspector\#Prevent_Ansel_from_being_injected.2Fenabled_for_a_specific_game "Link")

Using Nvidia Profile Inspector, is becomes possible to disable Ansel on a per-game basis if desired.

The display drivers loads Ansel related DLL files into all running applications they interface with by default, even if the application does not support or can make use of Ansel. This can cause issues for third-party tools or applications.

| Disable Ansel in the display driver profile for a specific game |
| --- |
| 1. Download and run the latest version of the Profile Inspector, see [Installation](https://www.pcgamingwiki.com/wiki/Nvidia_Profile_Inspector#Installation).<br>2. Using the **Profiles** list on top of the window, select the game-specific profile if one exists. If one does not exist, create a new one:<br>   <br>   1. Click on the **Create new profile** icon (the sun).<br>   2. Give the profile a relevant name. The name of the game is often used.<br>   3. Click on the **Add application to current profile** icon (a small window with an icon of binoculars with a "+" above it).<br>   4. Navigate to [<path-to-game>](https://www.pcgamingwiki.com/wiki/Glossary:Game_data#Installation_folder "Glossary:Game data") and select the game executable and click **Open**.<br>3. Scroll down to the **Other** section and configure **Enable Ansel** to `0x00000000 ANSEL_ENABLE_OFF`.<br>   <br>   - Setting may instead appear under **Other** section as **NVIDIA Predefined Ansel Usage**; set to `0x00000000 ANSEL_ALLOW_DISALLOWED`.<br>4. Finally click on **Apply changes** to save the changes. |

The built-in frame rate limiter of the Nvidia display drivers introduces 2+ frames of delay, while [RTSS](https://www.pcgamingwiki.com/wiki/Glossary:Frame_rate_(FPS)#Frame_rate_capping "Glossary:Frame rate (FPS)") (and possibly other alternatives) only introduces 1 frame of delay.[\[6\]](https://www.pcgamingwiki.com/wiki/Nvidia_Profile_Inspector#cite_note-6)

[AnandTech – Triple buffering: Why we love it _(archived)_](https://web.archive.org/web/20250716151454/https://www.anandtech.com/show/2794/2) — recommended reading about triple buffering/Fast Sync.

| Parameter | Description |
| --- | --- |
| **Flip Indicator** | Enables an on-screen display of frames presented using flip model. |
| **Frame Rate Limiter** | Enables the built-in [frame rate limiter](https://www.pcgamingwiki.com/wiki/Glossary:Frame_rate_(FPS)#Frame_rate_capping "Glossary:Frame rate (FPS)") of the display drivers at the specified FPS. |

| **GSYNC - Application Requested State** | If you know what this setting does, please add a description here. |
| **GSYNC - Application State** | Selects the technique used to control the refresh policy of an attached monitor.<br>- **Allow** enables the use of G-Sync, and synchronizes monitor refresh rate to GPUs render target.<br>- **Force off** and **Disallow** disables the use of G-Sync.<br>- **Fixed Refresh Rate** is the traditional fixed refresh rate monitor technology.<br>- **Ultra Low Motion Blur** (ULMB) uses back light pulsing at a fixed refresh rate to minimize blur.<br>It is highly recommended to change this using the **[Nvidia Control Panel](https://www.pcgamingwiki.com/wiki/Nvidia_Control_Panel "Nvidia Control Panel") \> Manage 3D Settings > Monitor Technology** instead to properly configured this and related parameters. |
| **GSYNC - Global Feature** | Toggles the global G-Sync functionality. It is recommended to change this using the **[Nvidia Control Panel](https://www.pcgamingwiki.com/wiki/Nvidia_Control_Panel "Nvidia Control Panel") \> Set up G-SYNC** instead to properly configured this and related parameters. |
| **GSYNC - Global Mode** | Controls whether the G-Sync feature will be active globally in fullscreen only, or in windowed mode as well. |
| **GSYNC - Indicator Overlay** | Enables a semi-transparent on-screen status indicator of when G-Sync is being used. If G-Sync is not being used, the indicator is not shown at all. |
| **GSYNC - Support Indicator Overlay** | Enables a diagnostics overlay which updates in real time. Can help determine which type of G-SYNC is being requested by a particular 3d application (is it requesting Exclusive Fullscreen or Windowed mode?) |
| **Maximum pre-rendered frames** | Controls how many frames the CPU can prepare ahead of the frame currently being drawn by the GPU. Increasing this can result in smoother gameplay at lower frame rates, at the cost of additional input delay.[\[7\]](https://www.pcgamingwiki.com/wiki/Nvidia_Profile_Inspector#cite_note-7) This setting does not apply to SLI configurations.<br>- Default is **Use the 3D application setting**, and it is recommended not to go above **3**. Values of 1 and 2 will help reduce input latency in exchange for greater CPU usage.[\[8\]](https://www.pcgamingwiki.com/wiki/Nvidia_Profile_Inspector#cite_note-8)<br>- When Vsync is set to `1/2 Refresh Rate`, a value of 1 is essentially required due to the introduced input latency.\[ _citation needed_\] |
| **Preferred Refresh Rate** | Controls the refresh rate override of the display drivers for games running in exclusive fullscreen mode. This also dictates the upper limit of the G-Sync range, as G-Sync will go inactive and the screen will start to tear at frame rates above the configured refresh rate, or V-Sync will kick in and sync the frame rate to the refresh rate if enabled.<br>- **Highest available** \- Overrides the refresh rate of the exclusive fullscreen game to whatever is the highest available on the monitor. If using G-Sync, frame rates above the requested refresh rate will result in screen tearing as G-Sync will go inactive, or V-Sync synchronizing the frame rate to the refresh rate if enabled.<br> This setting is only exposed in [Nvidia Control Panel](https://www.pcgamingwiki.com/wiki/Nvidia_Control_Panel "Nvidia Control Panel") for monitors supporting refresh rates of at least 100 Hz.<br> This setting is only effective in DX9/10/11 applications. On Windows 10 (and above) - [Fullscreen optimizations](https://www.pcgamingwiki.com/wiki/Windows#Fullscreen_optimizations "Windows") must also be disabled. |
| **Triple buffering** | Toggles triple buffering in **OpenGL** games. Enabling this improves performance when vertical sync is also turned on. |
| **Vertical Sync** | Controls vertical sync, or enables **Fast Sync**. Fast-Sync is essentially triple buffering for DirectX games, and is only applicable to frame rates above the refresh rate when regular V-Sync is disabled. |
| **Vertical Sync Smooth AFR behavior** | Toggles V-Sync smoothing behavior when V-Sync is enabled and SLI is active.<br>**Official description:**[\[9\]](https://www.pcgamingwiki.com/wiki/Nvidia_Profile_Inspector#cite_note-9)<br>Smooth Vsync is a new technology that can reduce stutter when Vsync is enabled and SLI is active.<br>When SLI is active and natural frame rates of games are below the refresh rate of your monitor, traditional Vsync forces frame rates to quickly oscillate between the refresh rate and half the refresh rate (for example, between 60 Hz and 30 Hz). This variation is often perceived as stutter. Smooth Vsync improves this by locking into the sustainable frame rate of your game and only increasing the frame rate if the game performance moves sustainably above the refresh rate of your monitor. This does lower the average framerate of your game, but the experience in many cases is far better. |
| **Vertical Sync Tear Control** | Toggles **_Adaptive_ V-Sync** when vertical sync is enabled. Not available when G-Sync is enabled.<br>**Official description:**[\[10\]](https://www.pcgamingwiki.com/wiki/Nvidia_Profile_Inspector#cite_note-10)<br>Nothing is more distracting than frame rate stuttering and screen tearing. The first tends to occur when frame rates are low, the second when frame rates are high. Adaptive VSync is a smarter way to render frames using Nvidia Control Panel software. Works on the entire image

Requires MSAA to work

Unknown settings

These are unknown settings or overrides that Nvidia Profile Inspector is not aware of what they determine or controls.

Most of these are game-specific overrides only applicable to a few games, or new flags that were introduced in newer versions of the display drivers.

Not all unknown settings can be properly exported and imported at will. If a built-in profile using one of those have been removed, the recommendation is to reinstall the display drivers to ensure they are restored fully.
